Contacts Import

Import many contacts into Easycloudbooks at once, saving you time when you're setting up your practice. You can import Contacts using a template XLS file or XLXS file. If you already have contacts in Easycloudbooks, you can import more.

Which are the Mandatory Fields in contact import ?

Contact Name :- First name and last name are Compulsory field. Note (already existing contact will not be imported).
Mobile Number :- Good to have Contact Phone Number for unique identification.
Optional Fields : Email ID, Contact Address, DOB, Passport No, Aadhar No, Married, Aniversary are optional fields.

Instruction and Rules for Combine Import !

If you want to add/update multiple contacts only then you can use contact.xlsx.



And if you want to add/update multiple Companies then for that use Company.xlsx.



And if you want to add contact and company and also want that contact should be linked with company or vice versa then use Combine.xlsx.



So, If you are willing to Import Contact and Company with linking then Open Combine.xlsx sheet.
Now to Import Companies, there are two fields that plays key role. 1st is Company Name and PAN. And same for the Contact Important fields are Contact Name and Mobile No.
If you have left Company name field blank then ECB will ignore other fields of the company and will check for Contact's field in same row and will do add or update operation on contact.
And as If you have filled Company name and other fields of the company and had left Contact name blank then ECB is going to ignore the Contact and will do add or update operation on company.
Now while importing a sheet, If ECB finds Company name that already exists there in the database then PAN will play an important role in such situation for identification of correct company.
So, we recommend to have unique PAN in front of every Company. And same role plays Mobile No for the Contacts.
EX:- Now, If you have that question that, what if I have two different clients with exact same name then will ECB allow me to Import both the clients or not?? The answer is YES, It is possible that two clients might have same name, but their PAN or Mobile No has to be different. So, two Contacts or two Companies can have same name but there mobile and PAN will be different. And on that basis, If ECB finds same PAN or Mobile No. exists then it will update the existing record with new record. And if, there are more than 1 duplicate PANs or Mobile Numbers exists then ECB will ignore that record.

Rules and Situations for Company Import with Contacts !

EasyCloudBooks provides 3 types for Import functionalities.
1. Company Import- which can be used for Creation of Companies only.
2. Contact Import- which can be used for creation of contacts only.
3. Combine Import- It will create Contact and Company with Linking.
There can be two situations while Importing Contacts and Company.
1. Import Company with Name and Without PAN.
2. Import Company with Name and PAN.

1. If you Import company without PAN.

Scene 1: If you import companies without PAN
, and same name doesn’t exists then EasyCloudBooks will create company.
, and same name exists then easycloudbooks will update the record.
So, EasyCloudBooks prefers to Import Companies with 'PAN' and Contacts with 'Mobile No'.

2. Now, If you Import company with PAN then there are 4 possibilities
1. Any PAN Doesn’t Exists
2. Same PAN Exists
3. Same PAN Exist more than one
4. Other PAN Exist

Scene 1: If any PAN Doesn’t Exists
, And same Name exists then easycloudbooks will Update Name and other fields.
, And same name doesn’t exists in the system then EasyCloudBooks will create New Company.
, And same name exists more than once then easycloudbooks will ignore that record.

Scene 2: If same PAN exists
, and same name exists then easycloudbooks will update those details in database.
, and the company name is different then easycloudbooks will update name and other details.
, and same name exists more than once then easycloudbooks will Ignore that record.

Scene 3: If same PAN exists more than once then easycloudbooks will not update the record but if it is possible then will do the linking.

Scene 4: If any other Pan Exists.
, and same name exists then easycloudbooks will create new Company.
, and same name doesn’t exists then easycloudbooks will create new company.
, and same name exists more than once then easycloudbooks will ignore that record.

Which file format supported by Easycloudbooks?

.xls or .xlxs formats supported by system.

What is the maximum size of file which is to be imported and how many contacts are imported at a time?

1000 contacts are imported at a time and Up to 2 mb size, file can be imported.

Any report is generated after completion of Contact import?

Yes, Report is generated after completion of import which contain
1.Successfully Imported Contacts Count
2.Already Existed Contacts Count
3.Error Contacts Count
4.File Rejection Message when file is not in proper format
5.User can check this Report in Documnents Vault or Work Task page.
