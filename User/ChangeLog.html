Version 5.0.02
27 April 2019
Modules Changes Description
Service Page
  • First change is automated project period. Friends we have been receiving requests from you all that when projects get automatically created from schedule than period should automatically appear instead of NONE and in this version we have released this feature. To make it work you need to do some changes in services settings. Here on services on which you have applied schedular you need to provide period selection setting. Period selection - basically asks you when project get created as per schedule it belongs to current month, previous month, next month etc. and when you provide this setting period will automatically appear in all the projects getting created with scheduler.
Project Page
  • Second change you will notice is on project page. On project page you will see a small count on each stage of project. Which indicates how many projects requires your urgent attention today and on which you need to follow up to client or your team or govt. offices.
Automation Page
  • Third change you will notice on automation page. Friends in the process of making it more easy to use when have shown automation in expanded view which will make you easily understand when automation gets triggered and what action is applied. Further to help you manage it we have given you filters like trigger even, apply to and action. so if you want to see only those automation where email action is applied you can filter action email and it will display all the automation where action is email.
Staff Online Status
  • Fourth change you will notice on top of screen is users status. So who is logged in from your team member and who is working on which project you can see it from here on click on all the screens.
Schedular Page
  • Fifth change is related to schedular naming. Friends when you create a new schedular or edit the old schedular, you will see name automatically appear. We suggest you to keep default name suggested by system because it will standardise it and your other team members will be stopped in creating duplicated scheduler.
Billing
  • Lastly friends I am very delighted to share that now we are totally on cloud which means you will experience greater speed. Further we have changed our billing from prepaid period based commitment to postpaid usage based monthly billing. which means you will not be charged for days when you don't use system. However if you have concerns about your over usage charge you can definitely set the budget limit for monthly usage. Know More

Version 2.1
Dec 19 2015
Modules Changes Description
Query Page
  • We are introducing new page for managing queries, whereby user will be able to reply on open queries for him.
  • Further query manager (person who is given this right on role page) will be able to view his queries as well as queries for other users to make him able to resolve queries faster.
Manage Role
  • Rights related to Query Page such as who can view query page, who can post query on query page, who is manager for query page
    (Query manager is person who is responsible for getting query solved from project managers as early as possible)
Project Module
  • Status note on project list page and on detailed project page now has facility to add next follow up date. Thereby status will turn red on those follow up dates and therefore nothing will be missed.
    Default sorting of the page will now be as per target date thereby it will automatically prioritize your projects.
    Note : you can anytime change the target date to sort your priorities.
Version 2.0
Dec 19 2015
Modules Changes Description
Login Section
  • We are introducing new feature of USERID, thereby users will be able to login with preferred userid (Non Changeable). Therefore existing users will need to create USERID.
Contact
  • In detailed page of any particular contact, we have activated email verify button and select primary checkbox. This will be used for communication and notification by your company related to that particular contact.
User Company
  • Now user can change his company profile with detailed information.
    (Note : Only owner or user who has such role privileges).
User Profile
  • User will be able to create his detailed profile, which will be visible to all the team members. This profile will be visible to all the companies in which he is user. Now onward user will be able to change his email id and maintain multiple email IDs.
  • User Need to compulsory maintain one primary email verified which will be used for communication in case user forgets his username or password.
Staff Management
  • A new column is added to record position of staff in user company. This can only be changed by owner or person with this right on role page.
  • A new tab is added named “Invites” for tracking invitation to new users for last one month.
  • Further for security concerns we are introducing compulsory change of password (8 characters with minimum one digit number) system on quarterly basis. Therefore existing users who have not changed their password for last three months will also need to change their password at launch of this version.
Manage Role
  • Under setting in manage role page you will be able to give rights on any role related to.
    1. Right to manage position of staff on manage staff page.
    2. Rights related to user company profile edit.
New User Company
  • Now user will be able to create new user company from top bar of change company menu.
Badges
  • Like “Contact Badges” and “Client Company Badges” which appears on mouseover on name, we are introducing “User Badges” which will appear on mouseover. This will display user profile details and details to visualise how busy the user is.
